# 场景/组件修改规则（最高优先级）

## 🔴 核心原则

**在修改场景物体、组件配置、预制体设置之前，必须：**

1. **先审视原有配置** - 完整了解当前设置的每个字段和值
2. **分析原有设计意图** - 理解用户为什么这样配置
3. **说明修改原因** - 解释为什么需要改变
4. **对比新旧差异** - 明确列出改动点和影响
5. **征求用户同意** - 等待用户确认后再执行

## 🚫 绝对禁止

- 未经分析就删除或清空组件字段
- 假设用户的配置是错误的
- 用"简化"为借口破坏原有功能
- 在不理解原有设计的情况下进行修改

## ✅ 正确流程

```
1. 发现问题
2. 读取并展示当前配置
3. 分析配置的设计意图
4. 提出修改方案（保留原有功能）
5. 说明：
   - 原有配置：XXX
   - 问题原因：XXX
   - 建议修改：XXX
   - 修改后效果：XXX
   - 对原有功能的影响：XXX
6. 等待用户确认
7. 用户自行修改或授权修改
```

## 📝 示例

**错误做法**：
```
发现 Toggle 显示异常 → 直接清空 graphic 和 transition
```

**正确做法**：
```
发现 Toggle 显示异常 → 
读取当前配置：
- 过渡 = 颜色色彩
- 目标图形 = Up_00 (Image)
- 图形 = Selected (Image)
- 分组 = Up (Toggle Group)

分析：这是一个完整的 Toggle 配置，用于实现槽位选中效果

问题分析：红色边框全部显示可能是因为...

建议方案：保留原有配置，通过 XXX 方式解决...

请确认是否同意此方案？
```

## 适用范围

- Unity 场景中的 GameObject
- 组件的 Inspector 配置
- 预制体设置
- ScriptableObject 配置
- 任何用户手动配置的内容
