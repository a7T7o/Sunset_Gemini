# 场景层级同步规则

## 核心原则

**每次对场景内物体进行新建、删除或修改组件时，必须同步更新对应的场景层级文档！**

## 文档位置

| 场景/区域 | 文档路径 |
|----------|---------|
| UI 系统 | `Docx/全局场景层级/UI层级结构.md` |
| 世界物体 | `Docx/全局场景层级/世界物体层级.md`（待创建） |
| 玩家 | `Docx/全局场景层级/玩家层级.md`（待创建） |

## 触发条件

以下操作必须同步更新文档：

1. **新建 GameObject** - 记录新物体的位置、组件、配置
2. **删除 GameObject** - 从文档中移除对应条目
3. **添加组件** - 记录组件名称和关键配置
4. **移除组件** - 从文档中移除对应条目
5. **修改组件配置** - 更新文档中的配置说明
6. **修改层级关系** - 更新文档中的层级结构

## 文档格式

```markdown
# [区域名称] 层级结构文档

## 更新规则

**每次对场景内 [区域] 物体进行新建、删除或修改组件时，必须同步更新本文档！**

## 最后更新

- **日期**: YYYY-MM-DD
- **更新者**: [名称]
- **更新内容**: [简述修改内容]

---

## [物体名称] 详细结构

```
物体名称 (GameObject)
├── 组件1 (Component)
├── 组件2 (Component)
│
├── 子物体1 (GameObject)
│   ├── 组件 (Component)
│   └── ...
│
└── 子物体2 (GameObject)
    └── ...
```

---

## 关键组件说明

### 组件名称
- 职责说明
- 关键配置
- 注意事项
```

## 检查清单

在提交代码前，确认以下事项：

- [ ] 是否修改了场景中的 GameObject？
- [ ] 是否添加/移除了组件？
- [ ] 是否修改了组件的关键配置？
- [ ] 对应的层级文档是否已更新？
- [ ] 文档中的"最后更新"信息是否已更新？

## 重要性

场景层级文档的作用：

1. **快速定位** - 无需打开 Unity 即可了解场景结构
2. **问题排查** - 快速确认组件配置是否正确
3. **知识传承** - 新成员可以快速了解项目结构
4. **避免重复错误** - 记录关键配置，避免遗忘
